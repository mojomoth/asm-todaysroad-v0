# SPEC — 오늘의길 (Today's Road) MVP

> 생성: `/debate-loop` · slug: `todays-road-v1` · 라운드 수: 6 · 수렴: FORCE_STOP(상한)

## 1. 목표
사용자가 현재 위치에서 가용 시간과 기분(감성)을 입력하면, 강남역 반경 1.5km 내에서
**검증된 산책 코스를 항상 5개 추천**하고, 선택 후 폰을 덜 보면서(음성 턴안내+단순 시각모드)
산책을 완주하게 한다. 코스 경로상 숨은 카페·식당도 함께 발견한다.

## 2. 배경 / 문제
- 늘 같은 산책길이 지겨운 거주자/여행객은 "지금 여기서 걷기 좋은 길"을 빠르게 찾기 어렵다.
- 기존 길찾기 앱은 목적지를 전제하며, 산책 중 계속 폰 화면을 봐야 한다.
- 토론 합의된 진짜 문제: **목적지 없이도 신뢰할 수 있는 코스를 즉시 얻고, 폰 의존을 줄여 걷는 경험**.
- MVP 핵심가치(재정의): "폰을 덜 보면서 걷기 좋은 길 속 숨은 카페·식당을 발견하는 산책 가이드".

## 3. 범위 (Scope)
- **In scope**
  - 강남역 반경 1.5km 핵심권역(5개 마이크로권역)
  - 큐레이션 고정 코스 초기 10~12개(파일럿 3개로 시작)
  - 현위치 + 가용시간(예: 15/30/45/60/90분) + 감성태그 기반 랭킹 추천(상위 K=5)
  - A-B 코스 + 템플릿 루프(수작업)
  - 지도 위 경로 표시(카카오 지도 JS SDK)
  - 음성on 턴안내(기본 프롬프트 ON) + 단순 시각모드(큰 화살표 + 남은거리)
  - 햅틱 2패턴(안내/도착)
  - 코스당 대표 카페/식당 3개(경로 1/3·2/3·종료, 50m내) — 큐레이션 메타데이터
  - 콜드스타트/위치권한 미허용/GPS·네트워크 오류/결과 fallback 화면
- **Out of scope (V1 이후)**
  - 카카오 로컬 API 실시간 POI 조회(법적 리스크) — 수작업 메타데이터로 대체
  - 루프 코스 자동 생성(NP-hard) — 템플릿만
  - 본격 햅틱 다패턴 + earcon(사용성테스트 통과 후)
  - UGC 코스, 사용자 리뷰/평점, 소셜 공유
  - 다국어(한글 우선)
  - 강남역 외 지역 확장, 실시간 혼잡도/날씨

## 4. 기능 요구사항
- FR-1 현위치 획득 후 강남역 1.5km 내 코스를 랭킹 정렬해 항상 상위 5개 노출(결과 0개 금지).
- FR-2 가용시간·감성태그를 입력 UI(시간 슬라이더 + 감성 칩 + CTA)로 받고, 미입력 시 기본값 적용.
- FR-3 랭킹 = 하드필터(반경) → 소프트 랭킹(시간 근접 점수 + 태그 매칭 자카드×객관축 1.5배) 정렬.
- FR-4 반경 fallback: 결과 부족 시 1→2→3→5→10km 단계 확대, 그래도 부족하면 태그 완화, K=5 보장(최소 3개).
- FR-5 선택 코스 경로를 지도에 표시하고 메트릭(거리·걸음수·예상시간) + 대표 POI 3개를 코스 카드에 표시.
- FR-6 산책 시작 시 음성 on/off 1회 프롬프트(기본 ON). 턴 30m 전 발화 + 단순 시각모드(화살표+남은거리).
- FR-7 GPS 약신호 구간은 단순 시각모드로 자동 전환(fallback).
- FR-8 POI 오류 신고 버튼 제공, 폐업/이전 신고 시 영업일 1~2일 내 검증·숨김.

## 5. 비기능 요구사항
- NFR-1 (성능) 앱 시작~첫 코스 추천 ≤3초, GPS 락온 ≤5초, 음성 턴안내 지연 ≤2초(사전생성 MP3 캐시 목표 ≤1초).
- NFR-2 (정확도) 음성 턴 감지 성공률 ≥90%, 맵매칭으로 GPS ±10m 보정.
- NFR-3 (운영) POI 메타데이터 월 1회 수동 갱신 + 사용자 신고 SLA(영업일 1~2일).
- NFR-4 (법무) 약관 면책조항("정보 제공 서비스, 정확성 미보장, 영업정보 변동 가능"), 위치 정보 처리방침 명시. — **법무 검토 필요(잔여)**.
- NFR-5 (접근성) 마커 색상+형태 이중구분, 아이콘 aria-label, WCAG AA 핵심 대비. (목표 5/10→상향은 V1)

## 6. 인터페이스 · 시임(Seam)
- 5모듈 경계: **Course**(GeoJSON 저장·랭킹 추천) / **Navigation**(GPS 수신·맵매칭·턴 감지) / **Voice**(사전생성 MP3 캐시·재생) / **POI**(큐레이션 메타데이터) / **MapAdapter**(카카오 지도 JS SDK 래핑).
- 외부 의존 격리: 지도·POI 소스는 어댑터/인터페이스로 추상화(교체 가능).
- 추천 엔진(반경 필터·태그 매칭)은 외부 호출 없는 순수 함수로 격리해 단위 테스트.
- 최상위 시임: `Navigation.onLocationUpdate(lat, lon, accuracy)` 단일 콜백으로 E2E 시뮬레이션 가능.
  - **주의(잔여 R6-6)**: 단일 콜백에 GPS(5~10Hz)+턴감지+TTS+렌더링이 묶이면 메인스레드 부하 → 저사양 실기기 측정 필요.

## 7. 도메인 용어
- **코스(Course)**: 경로 좌표(GeoJSON) + 메트릭 + 감성태그 + source(curated). 추천/네비/검색의 원자 단위.
- **A-B / 루프**: 코스의 토폴로지 속성(별개 엔티티 아님).
- **감성태그**: 객관축(거리 버킷/유형/시간대) + 주관 감성축(조용/활기/강변/야경/카페/골목 등). 배타쌍(조용↔활기), 공존 가능(강변+조용).
- **메트릭**: 경로좌표서 결정론적 재계산. 예상시간=거리/속도(평탄4.5·상향3·하향5 km/h)+고도보정(상승100m당+6분, 하강100m당-3분), 걸음수=거리(m)/0.68m. (잠정값, 베타 실측 ±15% 보정)
- **POI**: 코스 메타데이터로 부착된 대표 카페/식당(경로 50m내). 실시간 조회 아님.
- **추천 핀 / 현위치 마커 / 코스 카드**: 화면 구성 요소.

## 8. 수용 기준 (Acceptance Criteria) — 출시 게이트
- **MVP 게이트 (모두 충족 시 Go)**
  - AC-1 hit rate ≥75%: 코스당 3회 왕복 실측, 목표지점 도달 성공률(턴 누락 없음).
  - AC-2 사용성테스트(5인 현지): 4명 이상이 음성+시각만으로 코스 완주, 턴 놓침 ≤1회, 학습 5분 이내.
  - AC-3 음성 턴안내: 턴 30m 전 발화, 지연 ≤2초(GPS ±10m 범위).
  - AC-4 앱 시작~첫 코스 추천 ≤3초, GPS 락온 ≤5초.
  - AC-5 결과 0개 미발생(랭킹+fallback로 항상 ≥3개, 정상 ≥5개).
- **GA 게이트**
  - hit rate ≥90%, 사용성 95% 통과, 음성 지연 ≤1초, 코스 완주율 ≥70%.

## 9. 미해결 잔여 리스크 (FORCE_STOP — 출시 전 필수)
- **[R6-1 실행]** 게이트 검증 체크리스트의 담당·일정·출시일·롤백 기준 미확정. → 2주 내 확정.
- **[R6-2 기술]** TTS 엔진/라이선스/월 비용 미선택(음성은 핵심 기능). → Google TTS / Naver Clova / 사전녹음 비교 후 결정. 미정 시 현장검증(AC-2/AC-3) 자체가 불가.
- **[R6-3 검증]** 현장검증이 강남역 1지역·여름·대낮뿐. 겨울/우천/야간/타지역 GPS 성능 미검증 — 전국 일반화 불가.
- **[R6-5 법무]** 약관 면책조항의 한국 법 실효성 미검증(NFR-4). → 법무 자문 필수.
- **[R6-6 성능]** 단일 콜백 메인스레드 부하 미측정 → 저사양 실기기 부하 테스트.
- **[R6-9/R6-10 운영·개인정보]** GPS 약신호 시각모드 이탈률 미측정, 위치 히스토리 저장 여부·개인정보 처리방침 불명확.
- **[R1-10 접근성]** 5/10. 음성 안내 스크린리더 호환·WCAG AA 핵심은 V1.
- **[R5-4 스케일]** 큐레이션 3→15→100→500 로드맵은 기대치(리소스·자동화·지형데이터 미검증). 파일럿 3코스 후 재산정.
