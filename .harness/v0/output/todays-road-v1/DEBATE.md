# DEBATE — 오늘의길 (Today's Road)

> 생성: `/debate-loop` · slug: `todays-road-v1` · 라운드 수: 6 · 종료: FORCE_STOP(round=6 상한)

## 입력 기획 (원문 요약)

"오늘의길"은 사용자가 현재 위치에서 지금 걷고 싶을 때, 가용 시간과 취향(감성 태그)에 맞는
**목적지 없는 산책 코스**를 추천하는 서비스다. 길찾기 앱이 아니라 "지금 여기서 걷기 좋은 길"을
발견하는 산책 경험 서비스. 핵심 기능 7종: ①현위치 기반 추천 ②시간 기반 추천 ③루프·A-B 코스
④감성 태그 선택 ⑤지도 경로 표시 ⑥산책 중 음성 안내 ⑦경로 주변 카페·식당. 카카오 지도(웹) 사용,
전체화면 지도 + 상단 pill 헤더 + 코랄 추천핀 + 하단 코스 카드(제목·주소·태그·거리·걸음수·예상시간).

## 라운드 1

### 패널 발언 (요약)
- **[CEO]** 진짜 문제가 모호(신선함/시간/취향 3개 페인 혼재). 음성안내·카페추천은 자르기 후보. 코스 데이터 소스 불명. "폰 안 봄"과 선택 단계 폰 사용은 모순.
- **[Eng]** 코스 데이터 출처 미정(공공/큐레이션/생성), 백엔드 스택·공간인덱싱 미정, 루프 코스 생성은 NP-hard성 난제(카카오 Directions는 일반경로만), 추천 엔진 로직 불명, 음성 오프루트 임계값 미정.
- **[Design]** 입력 UI(시간/감성태그) 화면에 완전 부재, 네비 모드 화면 미정의, 카드 상호작용 모호, 접근성 4/10, 콜드스타트·오류 UI 부재.
- **[DevEx]** 위치권한 진입장벽(거절율 높음), 마법의 순간 2개 사이 15분 갭, "폰 안 봄 vs 지도 읽기 필수" 모순, 코스 신뢰 공백.
- **[PM]** 감성태그 정의·부여 주체 미정, 웹 음성안내 실현성(브라우저 닫으면 중단), 데이터 확보 방식 불명, MVP 지역 미정, 수용기준 수치 미검증.
- **[Domain]** "코스" 정의 중의성(고정 큐레이션 vs 실시간 생성)이 핵심 미결, 감성태그 온톨로지 미정, 걸음수/예상시간 계산식 미정, "목적지 없음" vs POI 추천 긴장.

### Devil 문제점
- R1-1 [critical] 코스 정의(존재 형태) 부재 → 전체 아키텍처 미정
- R1-2 [critical] "폰 안 봄" vs 선택·네비 필수 UX 모순
- R1-3 [critical] 추천 엔진 작동 원리 블랙박스
- R1-4 [critical] 루프 코스 생성 기술적 난제(NP-hard)
- R1-5 [critical] 데이터 소스·신뢰성·최신성 불명
- R1-6~R1-13 [major] 입력UI 부재 / 네비모드 화면 부재 / 감성태그 온톨로지 / 웹 음성 실현성 / 접근성 4/10 / MVP 지역 / 위치권한 장벽 / 15분 갭
- R1-14~R1-17 [minor] 콜드스타트 / 오류상태 UI / 여행객 언어 / 계산식

### Moderator 종합
- 합의: 현위치+시간 입력이 최소 실현 입력. 웹 플랫폼 타당. MVP는 A-B부터.

### 합의된 결론 (확정)
1. **제품 정의**: 강남역 반경 1.5km, 큐레이션 고정 산책 코스를 현위치+가용시간+감성태그로 추천. "폰을 덜 보면서 걷기 좋은 길과 숨은 카페·식당을 발견하는 산책 가이드".
2. **추천**: 하드필터(반경) + 소프트 랭킹(시간 근접·태그 매칭 자카드×객관축 1.5배), 항상 상위 K=5 노출(0개 없음), 반경 1→2→3→5→10km fallback.
3. **코스**: 단일 엔티티(source: curated). 메트릭은 경로좌표서 결정론 재계산. 예상시간=거리/속도(평탄4.5/상향3/하향5)+고도보정(상승100m당+6분/하강-3분), 걸음수=거리/0.68m. 잠정값, 베타 실측 ±15% 보정.
4. **POI**: 실시간 API 아님. 코스당 대표 카페/식당 3개(경로 1/3·2/3·종료, 50m내) 수작업 메타데이터. 월1회 갱신+신고버튼.
5. **음성/네비**: MVP=음성on 턴안내(기본 프롬프트 ON)+단순 시각모드. 햅틱 2패턴. 본격 햅틱+earcon은 사용성테스트 V1 게이트. 사전생성 MP3 캐시, 맵매칭 GPS±10m 보정.
6. **태그**: 하이브리드 온톨로지(객관축: 거리/유형/시간대 + 감성축), 배타쌍(조용↔활기)·공존축, 관리자 2인 검수.
7. **출시 게이트**: MVP hit≥75%·사용성 5인중 4인·음성지연≤2s·앱시작≤3s·GPS락온≤5s. GA hit≥90%·완주율≥70%.
8. **아키텍처**: 5모듈(Course/Navigation/Voice/POI/MapAdapter), 단일 시임 `onLocationUpdate(lat,lon,accuracy)`.
